## How to build
**sorry for my bad English QAQ**
1. create a hugo site
```bash
hugo new site <the name of your site folder>
cd <the folder's name>
```
2. use git
```bash
git init
# for my git version, branch main is called master, I need to rename it
git branch -m main
git submodule add -b main <the url of your favorite theme> themes/<name of your theme>
```
3. go to `theme/<name of your theme>/exampleSite/`, copy config/_default to `.`
4. get to config in `.`, edit `hugo.toml`, change the url, change the language
5. get to `./hugo.toml`, change url and lang too
6. create a repository and commit!!!!!
```bash
git add .
git commit -m "<what you wanna say>"
git push <url of your repository> main
```
7. get a github actions
go to action/pages, choose Deploy from a branch, change it to Github Actions
choose create your own workflow, the name is hugo.yaml, copy the text from hugo official website
https://gohugo.io/host-and-deploy/host-on-github-pages/
8. test!!!!!
9. Customize your blog!!!!!

## 咋搭？
看似简单实则一堆坑，有点恶心
1. 创建hugo site
```bash
hugo new site <site文件夹名>
cd <文件夹名>
```
2. 用git开个仓库
```bash
git init
# 注意你的branch叫啥，现在要叫main，如果还叫master之类的用下面这句改
git branch -m main
git submodule add -b main <主题的git网址> themes/<主题名字>
```
3. 把`theme/<主题名字>/exampleSite/`的config/_default复制到根目录
4. 把刚才复制过来的文件夹里的`hugo.toml`的url和语言改了
5. 把刚才根目录里的`hugo.toml`的url和语言也改了
6. 在github开个仓库，把本地文件传进去
```bash
git add .
git commit -m "<这次传文件目的是什么>"
git push <仓库网址> main
```
7. 让github给你搭
找到action/pages, 选择Deploy from a branch, 改成Github Actions
有一个自己创建actions还是workflow什么玩意, 进去之后会新建一个文件，名字改成hugo.yaml, 从下面这个官网抄
https://gohugo.io/host-and-deploy/host-on-github-pages/
9. 测试，上面我们只是设置了主题，里面啥都没设置
10. 自定义blog页面
